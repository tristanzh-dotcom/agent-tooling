# 技能名称：收工交接文档生成 (Handover Skill)
# 触发条件：当用户要求“交接”、“收工”时启用。

## 执行步骤（请严格按照以下步骤操作）：

1. **信息收集**：
   - 先确认权威项目根，禁止直接把当前工作目录（CWD）当成扫描锚点：
     - 优先读取当前目录及父目录中的 `AGENTS.md`，以其声明的 Scope、项目规则或明确路径作为项目根判断依据。
     - 如 `AGENTS.md` 不足以确认项目根，则使用当前对话指定的 handover 文件、最近的 `HANDOVER_*_YYYYMMDD.md` 或用户给出的项目路径定位权威项目根。
     - 如果候选项目根与 CWD 不一致，必须在继续前以权威项目根为准；如果仍无法唯一确认，先询问用户，不得跨目录扫描。
   - 只在已确认的权威项目根内扫描，找出今天修改过的核心文件；不得让 `rg`、`find`、`git` 等命令越界命中其他项目目录。
   - 回顾今天的对话，提取“已做出的决策”和“遇到的问题”。

2. **生成文档**：
   - 根据当前对话的核心业务主题，提取一个简短的英文标识（如 `turtle`, `mantou`, `web_auth`），在项目根目录生成名为 `HANDOVER_{主题标识}_YYYYMMDD.md` 的本地文件（将 `{主题标识}` 替换为英文标识，将 `YYYYMMDD` 替换为今天的日期）。
   - 示例：如果是处理龟生态的对话，文件名应为 `HANDOVER_turtle_20260530.md`。
   - 同时提取一个简短的中文主题名称，用于次日启动胶囊中的工作流锁定与对话重命名。
   - **【绝对禁止】：严禁创建或更新任何其他历史交接文档（如 HANDOFF.md、PROJECT_MEMORY.md、CHANGELOG.md 等）。你的输出目标有且只有这一个文件。**
   - 文件内容必须在文档开头强制包含以下板块，并将 `{当前生成的文件名}` 动态替换为实际生成的文件名，将 `{当前中文主题名称}` 动态替换为本次对话的中文主题名称：
     ````markdown
     ### ☀️ 次日启动胶囊 (Boot Prompt)
     请在明天开启新对话时，直接复制以下指令发给系统：

     ```text
     请静默读取并完全理解当前目录下的 `{当前生成的文件名}`。
     1. 请将本对话的逻辑分支锁定为：【{当前中文主题名称}】，并在你回复的第一句话使用 Markdown 的 H1 标题 (`# {当前中文主题名称} 工作流重启`) 输出，以便系统自动重命名此对话。
     2. 在执行任何操作前，请简要复述当前的【核心卡点】与【下一步行动】。等待我的确认后，再开始执行。
     ```
     ````
   - 文件内容必须严格包含以下 6 个模块，且“次日启动胶囊”必须排在最前：
     - **次日启动胶囊 (Boot Prompt)**：（提供明天新对话可直接复制的启动指令）
     - **第一性原理与项目上下文**：（简述当前功能的核心目的）
     - **今日完成事项**：（列出修改的文件和实现的逻辑）
     - **已作出的关键决策**：（为什么这么做，放弃了什么方案）
     - **未解决的风险/报错**：（留给明天排查的雷）
     - **下一步行动**：（明天接手第一步该看哪个文件，执行什么命令）

3. **最终确认**：
   - 当 `HANDOVER_{主题标识}_YYYYMMDD.md` 文件成功写入本地后，必须在【当前对话框】中输出以下严格格式的回复，并将 `{当前生成的文件名}` 替换为实际文件名，将 `{当前中文主题名称}` 替换为本次对话的中文主题名称，将 `文件绝对路径` 替换为实际绝对路径：
     ````markdown
     交接文档已生成至 [{当前生成的文件名}](文件绝对路径)，请确认。
     明天请在新对话中首先输入：
     ```text
     请静默读取并完全理解当前目录下的 `{当前生成的文件名}`。
     1. 请将本对话的逻辑分支锁定为：【{当前中文主题名称}】，并在你回复的第一句话使用 Markdown 的 H1 标题 (`# {当前中文主题名称} 工作流重启`) 输出，以便系统自动重命名此对话。
     2. 在执行任何操作前，请简要复述当前的【核心卡点】与【下一步行动】。等待我的确认后，再开始执行。
     ```
     ````
